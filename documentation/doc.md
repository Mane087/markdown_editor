# Description

[MD-Editor](https://mane087.github.io/markdown_editor/) is an open source project designed to facilitate the creation and editing of Markdown (.md) files.

The main objective of the project is to offer an intuitive and practical online tool that covers most of the common needs when documenting projects, writing technical guides, or creating content in Markdown.

The application is designed for both developers and anyone who needs to write clear, structured documentation without worrying about remembering all the syntax of the language.

![home_app](/public/imgs/home_app.png)

## Key features

MD-Editor offers an optimized editing experience through visual tools and keyboard shortcuts that streamline your workflow.
The editor features a set of interactive buttons that allow you to automatically insert common Markdown elements, such as:

- Headings
- Code blocks
- Images
- Links
- Lists
- Quotes, among others

This allows you to focus on the content without wasting time writing syntax manually.

![menu_app](/public/imgs/menu_app.png)
You can view the full details of the available features in the section: [editor options](/documentation/options_us.md).

## Shortcuts

In addition to insertion using buttons, MD-Editor includes keyboard shortcuts that allow you to perform the same actions quickly and efficiently, significantly improving productivity.

The complete list of shortcuts and how they work is documented in: [editor shortcuts](/documentation/options_us.md).

   ```bash
   git checkout -b feature/nueva-funcionalidad
   ```
3. Make your changes, ensuring that the code remains clean and readable.

5. Send a Pull Request explaining the purpose of your contribution.

### Before sending your PR, make sure that:

- The project compiles and works correctly.
- The change is aligned with the project's objective.
- The documentation has been updated if necessary.

## Reporting issues

If you find an error, unexpected behavior, or have a suggestion for improvement, you can report it by creating an issue in the repository.

When creating an issue, please include as much information as possible:

- Clear description of the problem.
- Steps to reproduce it.
- Expected behavior vs. actual behavior.
- Screenshots or examples (if applicable).

This helps identify and resolve the issue more quickly and effectively.

## Learning to use Markdown
